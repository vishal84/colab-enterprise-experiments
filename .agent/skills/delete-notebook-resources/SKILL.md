# Delete Notebook Resources

When assisting users with Jupyter notebook pipelines that connect to GCP, it is best practice to always provide a final execution block that cleans up the created resources. This skill should be applied whenever you update or create a notebook that spins up ephemeral cloud architecture.

## Identified Target Resources

Look out for the following standard resources when attempting to wipe a notebook's trace:

1. **GCS Staging Buckets**:
   - Variables like `BUCKET_NAME` or `STAGING_BUCKET`.
2. **Vertex AI Search (Discovery Engine)**:
   - Variables like `ENGINE_ID`, `DATA_STORE_ID`, `DATA_STORE_NAME`.
3. **Document AI Processors**:
   - Variables like `PROCESSOR_NAME`, `PROCESSOR_ID`.
4. **Vertex AI Endpoints**:
   - Variables like `ENDPOINT_ID`.

## General Strategy and Code

The cleanup code should be built into a function (e.g., `def clean_up_resources():`) and called underneath a commented execution check, preventing accidental wholesale deletion if the user hits "Run All".

### 1. Safely Deleting GCS Buckets
Buckets cannot be deleted natively if they contain files. Ensure your cleanup logic iterations over `bucket.list_blobs()` and purges them before dropping the bucket.
```python
storage_client = storage.Client(project=PROJECT_ID)
bucket = storage_client.get_bucket(BUCKET_NAME)
for blob in bucket.list_blobs():
    blob.delete()
bucket.delete()
```

### 2. Issuing Deletes to Vertex AI Search
The Python SDK for `discoveryengine` generates Long-Running Operations (LROs) when creating or destroying apps and datastores. To avoid complex operation waiting, executing standard authenticated REST requests is often preferred.

```python
creds, _ = google.auth.default()
auth_req = google.auth.transport.requests.Request()
creds.refresh(auth_req)
access_token = creds.token
headers = {"Authorization": f"Bearer {access_token}"}

# Trigger Engine Delete
requests.delete(
    f"https://{LOCATION}-discoveryengine.googleapis.com/v1alpha/projects/{PROJECT_ID}/locations/{LOCATION}/collections/default_collection/engines/{ENGINE_ID}",
    headers=headers
)
```

## Best Practices
- Place cleanup cells isolated at the very bottom of the notebook script.
- Catch `Exception` and print errors gracefully so that if a resource was already dropped or failed to be created initially, it does not throw Python tracebacks that stop subsequent resource destructors.
